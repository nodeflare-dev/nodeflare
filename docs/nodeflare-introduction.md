# Nodeflare - MCPをクローンせずに即座に試せる革命的ホスティングサービス

## はじめに：MCPを試すのが面倒すぎる問題

Model Context Protocol（MCP）は、AIエージェントに外部ツールやデータソースへのアクセス権を与える画期的なプロトコルとして注目を集めています。GitHubには日々新しいMCPサーバーが公開され、ファイル操作、データベース連携、API統合など、あらゆるユースケースに対応したMCPが増え続けています。

しかし、実際にこれらのMCPを試そうとすると、以下のような壁にぶつかります。

### 問題1: stdio形式のMCPがほとんど

GitHubで公開されているMCPサーバーの大半は**stdio形式**です。これはローカル環境でプロセス間通信を行う形式で、ターミナル上で動作させることを前提としています。

つまり：

- リポジトリをクローンする
- 依存関係をインストールする
- 環境変数を設定する
- ビルドして実行する

たった1つのMCPを試すだけでも、これだけの手順が必要になります。

### 問題2: ブラウザ版Claudeやmanusで使えない

開発作業以外の用途—例えば日常的な情報収集やタスク自動化—なら、ブラウザ版のClaude（claude.ai）やmanusで完結させたいところです。

しかし、stdio形式のMCPはローカルプロセスとして動作するため、ブラウザベースのAIアシスタントからは直接利用できません。SSE（Server-Sent Events）やWebSocket形式に対応していないMCPは、どれだけ優秀でもブラウザ環境では使い物にならないのです。

### 問題3: 自作MCPの認証実装が面倒

「自分でMCPを作ってみよう」と思い立っても、今度は別の問題が発生します。本番運用するなら認証機能は必須ですが、アクセストークンの発行・検証・失効管理、OAuthの実装、レート制限など、MCPのコア機能とは関係のないボイラープレートコードを大量に書く必要があります。

## 主要機能の詳細

### 1. マルチランタイム対応の自動ビルド＆デプロイ

Nodeflareは以下の技術スタックを自動認識し、適切なビルドパイプラインを実行します：

| ランタイム             | 対応状況                |
| ---------------------- | ----------------------- |
| Node.js / TypeScript   | npm, yarn, pnpm対応     |
| Python                 | pip, poetry, uv対応     |
| Go                     | 対応                    |
| Rust                   | 対応                    |
| Docker                 | カスタムDockerfile対応  |

リポジトリに含まれる`package.json`、`requirements.txt`、`Dockerfile`などを自動検出し、最適なビルド戦略を選択します。

### 2. stdio → SSE自動変換

ここがNodeflareの技術的なコアです。

従来のstdio形式MCPは、標準入出力を通じてJSON-RPCメッセージをやり取りします。Nodeflareのプロキシゲートウェイは、これをHTTPベースのSSE（Server-Sent Events）形式にリアルタイム変換します。

```
[クライアント]
    ↓ HTTPリクエスト（POST /mcp, Accept: text/event-stream）
[Nodeflare Proxy Gateway]
    ↓ JSON-RPC変換
[MCPサーバー (stdio)]
    ↓ 標準出力
[Nodeflare Proxy Gateway]
    ↓ SSEストリーム
[クライアント]
```

この変換は完全に透過的で、クライアント側からはネイティブのSSE対応MCPサーバーと同様に扱えます。ストリーミング対応なので、大量のデータを返す処理でもリアルタイムに結果を受け取れます。

### 3. 認証機能の自動付与

MCPサーバーを公開する際、認証なしでは誰でもアクセスできてしまいます。Nodeflareでは、**認証機能が最初から組み込まれています**。

#### Bearer Token認証

```bash
curl https://your-server.nodeflare.tech/mcp \
  -H "Authorization: Bearer mcp_xxxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"tools/list","id":1}'
```

アクセストークンはダッシュボードからワンクリックで生成できます。トークンにはプレフィックス`mcp_`が付与され、誤って他の認証情報と混同することを防ぎます。

#### OAuth 2.0対応

外部サービスやサードパーティアプリケーションと連携する場合は、OAuth 2.0による認証フローも利用可能です。

#### スコープベースの権限制御

トークンごとに細かな権限設定が可能です：

| スコープ                    | 説明                     |
| --------------------------- | ------------------------ |
| `*`                         | 全ての操作を許可         |
| `tools:*`                   | 全てのツール操作         |
| `tools:list`                | ツール一覧の取得のみ     |
| `tools:call`                | 全ツールの実行           |
| `tools:call:specific_tool`  | 特定ツールのみ実行       |
| `resources:read`            | リソースの読み取りのみ   |

例えば、外部に公開するAPIキーには`tools:list`と`tools:call:safe_tool`のみを許可し、内部用には`*`を付与する、といった運用が可能です。

### 4. シークレット管理

MCPサーバーには環境変数が必要なものも多くあります。例えばデータベース接続文字列やAPIキーなど、GitHubリポジトリにはコミットできない情報です。

Nodeflareでは、これらのシークレットをダッシュボードから安全に設定できます：

| 機能               | 説明                                   |
| ------------------ | -------------------------------------- |
| AES-256暗号化      | 保存時に暗号化                         |
| チーム共有         | ワークスペース内での安全な共有         |
| デプロイ時自動注入 | ビルド・実行時に環境変数として設定     |

### 5. 完全なアクセスログ

「誰がいつどのツールを呼んだか」を完全に記録します。

| 記録項目           | 説明                       |
| ------------------ | -------------------------- |
| タイムスタンプ     | リクエスト日時             |
| APIキーID          | 使用されたトークン         |
| ツール名           | 呼び出されたMCPツール      |
| レスポンスステータス | 成功/エラー               |
| 処理時間           | ミリ秒単位のレイテンシ     |
| エラーメッセージ   | 失敗時の詳細               |

企業での利用において、情報漏洩対策や監査対応として非常に重要な機能です。

## 具体的な利用イメージ

### シナリオ1: GitHubで見つけたMCPを30秒で試す

あなたはGitHubで興味深いMCPサーバーを見つけました。例えば「ローカルファイルシステムを操作できるMCP」や「Notionと連携するMCP」などです。

**従来のやり方:**

```bash
git clone https://github.com/example/notion-mcp
cd notion-mcp
npm install
export NOTION_API_KEY=your_key_here
npm run build
npm start
# さらにClaude Desktopの設定を変更...
```

**Nodeflareなら:**

1. Nodeflareダッシュボードで「New Server」をクリック
2. GitHubリポジトリURLを貼り付け：`https://github.com/example/notion-mcp`
3. 環境変数を設定：`NOTION_API_KEY=your_key_here`
4. 「Deploy」をクリック

数十秒後、あなたのMCPサーバーは `https://notion-mcp.nodeflare.tech/mcp` で稼働を開始します。ブラウザ版Claudeからすぐに使えます。

### シナリオ2: 自作MCPを安全に公開する

あなたは社内データベースにアクセスするMCPを開発しました。チームメンバーに使ってもらいたいのですが、認証なしで公開するのは危険です。

**Nodeflareでの公開手順:**

1. MCPのソースコードをGitHubにプッシュ（プライベートリポジトリでOK）
2. Nodeflareでサーバーを作成し、GitHub Appで連携
3. 認証を有効化（デフォルトでON）
4. チームメンバーごとにアクセストークンを発行
5. スコープを設定して権限を制御

これで、認証・認可・ログ記録を全て備えた本番グレードのMCPサービスが完成です。あなたが書くコードはMCPのビジネスロジックだけ。認証周りのコードは一切不要です。

### シナリオ3: 複数MCPを組み合わせたワークフロー

ある調査タスクを自動化したいとします：

1. **Google検索MCP**: キーワードで情報を検索
2. **WebスクレイピングMCP**: 検索結果のURLからコンテンツを取得
3. **Notion MCP**: 取得した情報をNotionに保存

従来なら3つのMCPをローカルで動かし、それぞれの設定を管理する必要がありました。

**Nodeflareなら:**

各MCPを個別のサーバーとしてデプロイし、一つのワークスペースで管理できます。

```

├── web-scraper-mcp      → https://web-scraper.nodeflare.tech/mcp
└── notion-sync-mcp      → https://notion-sync.nodeflare.tech/mcp
```

各サーバーの統計情報、アクセスログ、エラー率を一元的に監視できます。
